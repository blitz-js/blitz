<h1 align="center">The Fullstack React Framework</h1>

<h3 align="center">Built on Next.js ‚Äî Inspired by Ruby on Rails ‚Äî New Fullstack Data Layer</h3>
<h3 align="center"><a href="https://blitzjs.com" target="_blank">Read the Documentation</a></h3>
<br>

New ‚Äúno-API‚Äù data layer **lets you import server code into your React components** instead of having to fetch from an API.

Includes everything you need for production apps. **Everything end-to-end from the database to the frontend.**

Blitz brings back the **simplicity and conventions** of server-rendered frameworks like Ruby on Rails while preserving everything we love about React and client-side rendering!

<br>

### Quick Start

You need Node.js 12 or newer

#### Install Blitz

Run `npm install -g blitz`

#### Create a New App

1. `blitz new myAppName`
2. `cd myAppName`
3. `blitz start`
4. View your baby app at http://localhost:3000

<br><br>

![Architecture diagram](https://blitzjs.now.sh/img/architecture-diagram.png)

<br><br>

**Features:**<br>
‚ö°Ô∏è Built on Next.js<br>
‚ö°Ô∏è Don't have to build an API for client-side rendering<br>
‚ö°Ô∏è Client-side rendering, Server-side rendering, and fully static pages all in the same app<br>
‚ö°Ô∏è Full Typescript support with static, end-to-end typing (no code generation step needed like with GraphQL)<br>
‚ö°Ô∏è React Concurrent Mode enabled<br>
‚ö°Ô∏è Database/ORM agnostic, but Prisma 2 is default<br>
‚ö°Ô∏è CLI with code scaffolding, Rails-style console REPL, etc<br>
‚ö°Ô∏è GraphQL Ready<br>
‚ö°Ô∏è Deploy serverless or serverful<br>

**Other key features coming:**<br>
‚ö°Ô∏è Highly secure authentication <br>
‚ö°Ô∏è Authorization you can use on both server and client<br>
‚ö°Ô∏è Model validation you can use on both server and client<br>
‚ö°Ô∏è Plugins for easily adding libraries like Tailwind, CSS-in-JS, etc.<br>
‚ö°Ô∏è React native support<br>
‚ö°Ô∏è GUI so you don't have to use the CLI<br>

<br>

### The Foundational Principles

1. Fullstack & Monolithic
2. API Not Required
3. Convention over Configuration
4. Loose Opinions
5. Easy to Start, Easy to Scale
6. Stability
7. Community over Code

[The Blitz Manifesto](https://blitzjs.com/docs/manifesto) explains these principles in detail.

<br>

### What is Blitz Designed For?

Blitz is designed for tiny to large database-backed applications that have one or more graphical user interfaces.

While we currently only support web, we are pursuing the dream of a single monolithic application that runs on web and mobile with maximum code sharing and minimal boilerplate.
